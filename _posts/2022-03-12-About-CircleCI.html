About CircleCI

CircleCI is a program that combines continuous integration (CI) and continuous deployment (CD) to quickly allow developers to modify files and have them automatically deployed to their server for clients to see on a webpage.

Config File

The config file controlls most of the process, it allows the user to modify:

  • Build settings/process
  • IP
  • SSH Fingerprint
  • Workspace persistance
  • Deploy sttings/process

These processes are all looked at when the file is sent in to github and are completed in the specified order. The progress can be viewed on the CircleCI dashboard and if a problem comes up you can speciffically see what process caused the problem.

Persisting datat

Because data in the build job isn't avaliable in the deploy job the data from the build job needs to be persisted to the deploy job. This is done by adding, - persist_to_workspace: root: ./ (Absolute root of the project) paths: - _site (Folfer/files that you want to be persisted relative to the root), and in the deploy job you need to add, - attach_workspace: at: ./ (Root)

Conclusion

CircleCI is a very useful piece of software that greatly speeds the process of adding files to a server when you submit them to github.
